Instrument Reading Attribute Table

The Instrument Reading attribute table was created to store data associated with the instrument readings.

This involved creating a DB view, joining multiple tables to grab all the data required for the attribute table. A data model was also created for the view. GET and POST API end-points were created to fetch and update the table as readings were added, updated and removed from the table. Linq queries were used to filter the readings based on the client viewing the data.

Quick actions were added to zoom to the instrument locations, view/edit the detailed readings through the form, filter the table based on the columns and download the readings selected as PDFs.

Data seeding for the table was performed using Entity Framework migrations.

Add New Inspection Modal Popup

This is a modal popup for when a user wants to add new inspections to a strain feature. The user here has the option to either manually fill in all data through the form, or they can copy over data from a previously created inspection at the same or different strain feature.

The modal has a checkbox a user can check if they want to copy data from another inspection. An end-point was implemented to copy over the data from the source inspection to the target inspection. The modal has dropdowns for the target and source strain features, which users can filter by keyword. This functionality was added with the help of an end-point to filter the strain features by keyword. The modal also has a list of inspections that a user can copy data from. This list is populated based on the source strain feature selected by the user.

Enabling Instrument Module

We had been working on developing a new Instrumentation Module for our clients. We had been developing this on a demo client and when all the features were ready to go, I was tasked with enabling it for 5 of our clients.

This involved running a tool created by our team lead, to associate all our instruments with a new Instrument Location, based on their coordinates. A new Instrument Location layer was then enabled for these 5 clients through SQL scripts. Data seeding was done through EF migrations, to enable the tables and forms for these clients.

Refactoring Enum Controller

Our Instrumentation Enums Controller had a separate end-point for each of the enums, and there were a lot! I updated our controller to have one generic end-point for all our enums. This dramatically reduced the size of our controller and also saved a lot of time for devs creating new enums, since they no longer have to create a new end-point for every enum.

Personal Budget API

August 2022

This is a REST API that is used to keep track of an individual's personal budget. A user can add categories to/remove categories from their budget and assign a balance amount to each category.

Any debit or credit is reflected on the remaining balance. We also have a separate table of transactions that keeps track of any and all transactions between the categories of our budget.

We have two separate tables - one for the personal budget with its different categories, and one for the transactions between these categories with their dates and amounts. Any change in one of these tables is reflected on the other.

The database, with these tables, is deigned using PostgreSQL and the server that handles the HTTP requests is designed using Express. The server and database are connected using node-postgres and this API is documented using Swagger API.

Languages: Node.js, Express, SQL

Image Partitioning Trees

March 2022

This application, implemented in C++, uses a binary tree (PTree) to represent an image. The nodes of this tree represent rectangular regions of a PNG image.

Operations on this tree allow us to Prune the tree which results in a more pixelated image. The Prune function attempts, starting near the top of a freshly built tree, to remove all the descendants of a node, if all of the leaf nodes below the current node have colour within tolerance of the node's average colour. This application also allows us to flip an image both horizontally and vertically. The effects of these operations can be observed using the render function which renders an image using the root node of a tree as an input.

Languages: C++
